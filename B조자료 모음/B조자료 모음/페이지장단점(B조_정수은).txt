## 바라봄

https://barabom.me/

- 장점
    - 소개 페이지  : 스크롤에 따라 1개의 화면으로 분리되어 한눈에 확인 가능한 구조
    움직임이 추가되어 디자인적 요소가 뛰어남
    
    - 진행바(최상단) : 바라봄 소개 페이지에서 스크롤 이동 및 카테고리 이동 시 페이지 최상단에  주황색 로딩바가 출력됨 (현재 페이지에서의 스크롤 진행도 / 페이지를 불러오기까지의 로딩 시간 확인 가능함 (둘 중 하나의 기능으로 통일하면 좋을듯))
    - 앱 서비스 기반이다보니 웹에서 사용 가능한 기능이 딱히 없음
    웹 기준으로 만든다면 소개 페이지에서 각 카테고리로 이동 가능하면 어떨지?
    - 연혁 카테고리에서 글으로만 작성된게 아닌 타임 테이블 별로 아이콘 및 이슈 내용을 그림과 함께 삽입하여 디자인 요소 및 직관적 확인 가능

- 단점
    - 아이콘 옆에 페이지 이름을 기재하여 서비스 이름을 더 명확하게 전달하는게 좋을 것 같음
    기존 웹 페이지들의 통상적인 위치이며, 현재는 카테고리에 바라봄소개, 바라봄연혁 식으로 기재되어 중복되는(?) 느낌이 듬
    - 놀러가요 카테고리에 여행, 카페, 유치원 등이 한 페이지 내에서 구분없이 업로드되어있는데 카테고리 별로 세분화하면 좋을 듯
    - 마이 페이지 버튼이 일반적인 사용자 UI의 형태가 아니며 앱 기반이어서인지 개인 정보 확인 및 수정이 불가함
        
- 추가 사항
    - 페이지 최상단 이동 버튼 추가
    - 검색 기능 추가

## 라이펫

https://lifet.co.kr/

- 장점
    - 상담 버튼으로 접근성 용이
    - 귀여운 아이콘과 파스텔 톤 색감으로 부드러운 느낌

- 단점
    - 웹 메인 페이지와 건강관리 메인 페이지가 동일
    웹 메인에서는 해당 웹페이지에 대한 설명, 안내로 꾸미면 어떨지
    - 스토어에서 4개의 상품을 미리보기 할 수 있는데 <>버튼 or 좌,우 스크롤으로 추가적인 상품 더 볼 수 있으면 좋을거 같음
    - 반려동물 전환 버튼이 더 명확하게 클릭 체인지 형태로 직관적이지 않음
    - 건강관리 메인 페이지에서 마우스 형태 전환만 있고 호버 기능이 없어 링크 이동이 가능한 페이지인지 바로 알기 어려움
    - 강아지 검사 점수가 어떤 기능을 하는건지 바로 알기 어려움
